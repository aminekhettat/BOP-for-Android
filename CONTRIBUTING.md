# Contribution au projet BOP for Android

Merci de l'intérêt que vous portez à l'amélioration de BOP ! En tant que projet axé sur l'accessibilité, nous avons quelques règles pour assurer la qualité.

## Comment Contribuer
1. **Signaler des bogues** : Utilisez les "Issues" GitHub en décrivant précisément le comportement attendu et observé.
2. **Proposer des fonctionnalités** : Ouvrez une "Issue" pour discuter des nouvelles idées avant de commencer le développement.
3. **Pull Requests** : 
   - Créez une branche thématique.
   - Assurez-vous que les tests passent (`./gradlew test`).
   - Documentez vos changements avec KDoc.
   - Vérifiez l'accessibilité TalkBack pour tout nouvel élément UI.

## Standards de Code
- Suivez les conventions Kotlin officielles.
- Utilisez des noms de variables explicites.
- Documentez les fonctions publiques.

## Engagement envers l'Accessibilité
Toute contribution graphique DOIT être accompagnée d'une description sémantique appropriée pour les lecteurs d'écran.
